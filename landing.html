Local inference

MLX on Apple Silicon — blazing fast, fully private.

The built-in FastAPI server exposes an OpenAI-compatible /v1/chat/completions endpoint with streaming SSE. Drop in any MLX model from ~/.lmstudio/models/.

  • ✓ Streaming token generation via SSE
  • ✓ Vision support with MLX VLM models
  • ✓ OpenAI-compatible — works with any client
  • ✓ Python 3.12 + FastAPI backend
# FastAPI MLX inference server
from fastapi import FastAPI
from mlx_lm import load, generate

app = FastAPI()
model, tokenizer = load("~/.lmstudio/models/qwen")

@app.post("/v1/chat/completions")
async def chat(req: ChatRequest):
  return StreamingResponse(
    generate_stream(model, req),
    media_type="text/event-stream"
  )

Spec-driven workflow

From idea to implementation, step by step.

Specs formalise the development process so the agent always knows what to build, why, and how — with you in control at every stage.

1
Requirements
Define what you want to build in plain language.
2
Design
The agent proposes architecture and data models.
3
Tasks
Work is broken into a DAG of executable tasks.
4
Implementation
Agents execute tasks autonomously, with your oversight.
// tasks.md — DAG task graph

const tasks = [
  { id: "explore", deps: [] },
  { id: "gather-ctx", deps: ["explore"] },
  { id: "implement", deps: ["gather-ctx"] },
  { id: "test", deps: ["implement"] },
];

// Orchestrator resolves execution order
await orchestrator.run(tasks);

// ✓ explore
// ✓ gather-ctx
// ✓ implement
// ✓ test
